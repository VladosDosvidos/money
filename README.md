# 💸 Друже, де мої гроші? | Калькулятор втраченої вигоди від боргу

Елегантний, мінімалістичний та саркастичний односторінковий калькулятор альтернативної вартості боргу. Сайт розраховує, який чистий прибуток міг би отримати кредитор, якби замість надання безвідсоткової позики другу вклав ці кошти в один із консервативних фінансових інструментів в Україні.

---

## ✨ Особливості проекту

* **Динамічний калькулятор**: користувач вводить будь-яку суму боргу та обирає точну дату видачі за допомогою календаря.
* **Аналіз тривалості**: скрипт автоматично вираховує тривалість прострочення у роках, місяцях та днях.
* **6 Варіантів заощаджень** (1 базовий депозит + 5 консервативних інструментів):
  1. **Депозит у гривні (базовий сценарій)** — середня чиста річна ставка (після податків 19.5%): **10.78% APY net**.
  2. **ОВДП (Гривня)** — державні військові облігації, звільнені від податків: **16.00% APY net**.
  3. **Готівковий Долар (USD)** — пасивне збереження за рахунок росту курсу НБУ з 41.50 до 44.23 грн/$: **8.77% APY**.
  4. **Валютний депозит / Стейкінг USDT** — купівля валюти + 6% річних нарахувань: **15.04% APY**.
  5. **Індекс S&P 500 (Акції США)** — інвестування у світову економіку (ріст індексу + валютний ріст): **30.10% APY**.
  6. **Золото (XAU)** — шалений ріст ціни на золото на світових ринках за цей період: **61.38% APY**.
* **Елегантний UI**:
  * Темна футуристична тема з м'якими неоновими та золотими акцентами.
  * Витончений ефект матового скла (**Glassmorphism**) з динамічним розмиттям фону.
  * Плавна мікро-анімація збільшення/зменшення чисел (**Count-Up** за допомогою `requestAnimationFrame`).
  * Інтерактивна шкала жалю ("Смирення" ➔ "Тривога" ➔ "Лють") з мінливими саркастичними текстовими коментарями.
* **Функція «Надіслати натяк другу»**: кнопка автоматичного копіювання в буфер обміну ввічливого, але переконливого повідомлення для боржника із детальними розрахунками його «безкоштовного» кредиту.

## 🛠️ Технологічний стек

Сайт реалізований як **статичний Single Page Application (SPA)** без використання важких JS-фреймворків чи CSS-бібліотек. Це забезпечує:
1. **Миттєве завантаження** (100 балів у Google Lighthouse).
2. **0% зовнішніх залежностей** (не потребує `npm install` для роботи клієнтської частини).
3. **Високу надійність та безпеку** (чистий ванільний JS, семантичний HTML5, сучасний CSS3 з Custom Properties).

## 🚀 Як запустити локально

Проект готовий до роботи одразу після клонування. Оскільки він складається з чистих статичних файлів, ви можете запустити його будь-яким зручним способом:

1. **Спосіб 1: Live Server (VS Code)**
   * Відкрийте папку проекту у VS Code.
   * Натисніть правою кнопкою миші на `index.html` та оберіть **Open with Live Server**.

2. **Спосіб 2: Через Node.js (локальний сервер)**
   * Перейдіть у папку проекту в терміналі.
   * Запустіть швидкий сервер без встановлення:
     ```bash
     npx local-web-server
     ```
   * Або використовуйте будь-який інший сервер, наприклад `http-server`:
     ```bash
     npx http-server ./
     ```

## 🌎 Інструкції з безкоштовного деплою

Сайт статичний, тому його розміщення в мережі є повністю безкоштовним.

### Варіант 1: Firebase Hosting (Рекомендовано)
1. Переконайтеся, що встановлено Firebase CLI:
   ```bash
   npm install -g firebase-tools
   ```
2. Авторизуйтесь у своєму Google-акаунті:
   ```bash
   firebase login
   ```
3. Створіть новий проект у [консолі Firebase](https://console.firebase.google.com/).
4. Ініціалізуйте хостинг у папці проекту:
   ```bash
   firebase init hosting
   ```
   * *Виберіть створений проект.*
   * *На запитання "What do you want to use as your public directory?" вкажіть `.` (поточна папка).*
   * *На запитання "Configure as a single-page app?" оберіть `No` (або `Yes`, це не критично).*
   * *На запитання про перезапис `index.html` оберіть `No` (важливо!).*
5. Опублікуйте сайт:
   ```bash
   firebase deploy
   ```

### Варіант 2: Vercel (Найпростіший для GitHub-репозиторіїв)
1. Залийте код проекту на свій GitHub.
2. Зайдіть на [Vercel](https://vercel.com/) та авторизуйтесь через GitHub.
3. Натисніть **Add New Project**, виберіть свій репозиторій.
4. Натисніть **Deploy**. Vercel автоматично розгорне сайт та надасть безкоштовний SSL-домен (наприклад, `my-debt-tracker.vercel.app`).

### Варіант 3: GitHub Pages
1. Створіть репозиторій на GitHub та залийте файли проекту (`index.html`, `styles.css`, `app.js`).
2. Перейдіть у вкладку **Settings** репозиторію.
3. Оберіть розділ **Pages** у лівому меню.
4. У блоці **Build and deployment** у випадаючому списку гілок оберіть `main` (або `master`) та папку `/ (root)`.
5. Натисніть **Save**. Через 1-2 хвилини ваш сайт з'явиться за адресою `https://<ваше-ім'я-користувача>.github.io/<назва-репозиторію>/`.
